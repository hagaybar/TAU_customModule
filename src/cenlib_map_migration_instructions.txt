 ---
 Implementation Steps

 1. Update CSV Data Source

 Find: The current Google Sheets fetch URL/service

 Replace with:
 const CSV_URL =
 'https://d3h8i7y9p8lyw7.cloudfront.net/data/mapping.csv';

 async loadMappingData(): Promise<LocationMapping[]> {
   const response = await fetch(CSV_URL);
   if (!response.ok) {
     throw new Error(`Failed to load mapping CSV:
 ${response.status}`);
   }
   const csvText = await response.text();
   return this.parseCSV(csvText);
 }

 2. Update SVG Map Loading

 Find: The bundled SVG assets or current loading mechanism

 Replace with dynamic loading:
 const MAPS_BASE_URL = 'https://d3h8i7y9p8lyw7.cloudfront.net/maps';

 async loadFloorMap(floor: number): Promise<string> {
   const url = `${MAPS_BASE_URL}/floor_${floor}.svg`;
   const response = await fetch(url);
   if (!response.ok) {
     throw new Error(`Failed to load floor ${floor} map:
 ${response.status}`);
   }
   return response.text();
 }

 3. SVG Highlighting Integration

 The SVG files contain elements with IDs matching the svgCode column
 in the CSV:

 highlightLocation(svgContainer: HTMLElement, svgCode: string): void {
   const element = svgContainer.querySelector(`#${svgCode}`);
   if (element) {
     element.classList.add('highlighted');
     // Or apply inline styles for highlighting
   }
 }

 4. Bilingual Support

 Use the appropriate column based on user's locale:
 - English: libraryName, collectionName, description, shelfLabel,
 notes
 - Hebrew: libraryNameHe, collectionNameHe, descriptionHe,
 shelfLabelHe, notesHe

 ---
 CORS Configuration

 The CloudFront distribution is configured to allow requests from:
 - https://tau.primo.exlibrisgroup.com (Production Primo)
 - http://localhost:4200 (Development)
 - http://localhost:4201 (Alternative dev)

 No authentication required for public asset access (CSV and SVG
 files).

 ---
 Caching Considerations

 - CloudFront caches assets for up to 1 hour (3600 seconds)
 - Consider adding cache-busting if real-time updates are needed:
 const url = `${CSV_URL}?t=${Date.now()}`;
 - Or use appropriate Cache-Control headers in fetch requests

 Error Handling Recommendations

 async loadAssets(): Promise<void> {
   try {
     const [csvData, svgMaps] = await Promise.all([
       this.loadMappingData(),
       Promise.all([0, 1, 2].map(floor => this.loadFloorMap(floor)))
     ]);

     this.mappings = csvData;
     this.floorMaps = svgMaps;
   } catch (error) {
     console.error('Failed to load shelf map assets:', error);
     // Show user-friendly error message
     // Optionally fall back to cached data if available
   }
 }
